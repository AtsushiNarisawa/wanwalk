## ✅ 改善点の詳細分析

### 改善1: おすすめエリアの表示方式

#### **変更前（現在）**
```
カルーセル表示
→ スワイプして閲覧
→ タップ → エリア一覧画面（二度手間）
```

#### **変更後（提案）**
```
エリアカード3枚を縦表示
→ すぐに3つのエリアが見える
→ 「一覧を見る」ボタン → エリア一覧画面
```

**メリット**:
- ✅ 3つのエリアが一目で確認できる
- ✅ スワイプ操作が不要
- ✅ 「一覧を見る」で明確な導線
- ✅ 縦スクロールのみで全コンテンツ閲覧

**実装イメージ**:

### 改善2: 日常の散歩タブのランディング画面

#### **変更前（現在）**
```
日常の散歩タップ
→ すぐにGPS記録開始（DailyWalkingScreen）
```
**問題点**:
- ❌ 誤タップで散歩が開始される
- ❌ 過去の日常散歩履歴が見られない
- ❌ ユーザーの意図を確認できない

#### **変更後（提案）**
```
日常の散歩タップ
→ ランディング画面表示
   - 「散歩開始」ボタン → GPS記録開始
   - 「散歩記録」ボタン → 日常散歩の履歴表示
```

**メリット**:
- ✅ 誤操作防止
- ✅ 明確なアクション選択
- ✅ 日常散歩の履歴にアクセス可能
- ✅ ユーザーの意図を尊重

**実装イメージ**:
```dart
// 新しいランディング画面
class DailyWalkLandingScreen extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text('日常の散歩')),
      body: Padding(
        padding: EdgeInsets.all(24),
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: [
            // 散歩開始ボタン
            ElevatedButton(
              style: ElevatedButton.styleFrom(
                minimumSize: Size(double.infinity, 120),
                backgroundColor: Colors.green,
              ),
              onPressed: () => Navigator.push(
                context,
                MaterialPageRoute(
                  builder: (_) => DailyWalkingScreen(),
                ),
              ),
              child: Column(
                mainAxisAlignment: MainAxisAlignment.center,
                children: [
                  Icon(Icons.play_circle, size: 48),
                  SizedBox(height: 12),
                  Text(
                    '散歩を始める',
                    style: TextStyle(fontSize: 20),
                  ),
                ],
              ),
            ),
            
            SizedBox(height: 24),
            
            // 散歩記録ボタン
            OutlinedButton(
              style: OutlinedButton.styleFrom(
                minimumSize: Size(double.infinity, 100),
              ),
              onPressed: () => Navigator.push(
                context,
                MaterialPageRoute(
                  builder: (_) => DailyWalkHistoryScreen(),
                ),
              ),
              child: Column(
                mainAxisAlignment: MainAxisAlignment.center,
                children: [
                  Icon(Icons.history, size: 40),
                  SizedBox(height: 12),
                  Text(
                    '散歩記録を見る',
                    style: TextStyle(fontSize: 18),
                  ),
                ],
              ),
            ),
          ],
        ),
      ),
    );
  }
}
```

## 🔧 実装の前提条件

### 必須データ/API

1. **最新ピン投稿取得RPC**
   - 関数名: `get_recent_pins`
   - 引数: `p_limit: INTEGER (default 10)`
   - 戻り値: ピン投稿リスト（写真URL、ユーザー名、エリア、いいね数）

2. **ピン投稿データの存在**
   - `route_pins`テーブルに写真付きピンが存在すること
   - Week 3データ（箱根15ルート、横浜10ルート、鎌倉10ルート）にピン投稿が含まれること

3. **日常散歩履歴画面**
   - 既存の`WalkHistoryScreen`を日常散歩専用にフィルタリング
   - または新規画面`DailyWalkHistoryScreen`を作成
