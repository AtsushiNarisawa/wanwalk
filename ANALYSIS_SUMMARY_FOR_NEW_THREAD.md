## 💡 核心的価値提案

### なぜWanMapが必要なのか？

**ユーザーの課題**:
- 🐕 新しい散歩コースを探したい
- 📸 愛犬との素敵な瞬間を共有したい
- 🗺️ 地域の愛犬家と繋がりたい
- 🏅 散歩のモチベーションが続かない

**WanMapの解決策**:
1. **GPS記録による散歩ルート自動生成** - リアルタイムで距離・時間・ルートを記録
2. **ソーシャル機能** - ピン投稿、いいね、フォロー、コメント
3. **ゲーミフィケーション** - バッジシステムによるモチベーション向上
4. **地域密着型コミュニティ** - 箱根エリアを中心とした愛犬家ネットワーク
5. **DogHub箱根連携** - 公式おすすめコースの提供

## 🏗️ 技術アーキテクチャ

### フロントエンド
- **Flutter 3.0+**: クロスプラットフォームUIフレームワーク
- **Riverpod 2.6**: 状態管理（Provider + AutoDispose対応）
- **Flutter Map**: OpenStreetMap/MapLibreベースの地図表示
- **CachedNetworkImage**: 画像キャッシュによるパフォーマンス最適化

### バックエンド (Supabase BaaS)
- **PostgreSQL**: リレーショナルデータベース
- **Supabase Auth**: JWT認証（Email/Password）
- **Supabase Storage**: 画像保存
- **Supabase Realtime**: リアルタイム通知

### プロジェクト構造
```
lib/
├── screens/      (39ファイル) - UI画面
├── providers/    (24ファイル) - Riverpod状態管理
├── services/     (21ファイル) - ビジネスロジック
├── models/       - データモデル
├── widgets/      - 共通ウィジェット (Shimmer, Error, Empty等)
├── config/       - デザインシステム (Colors, Typography, Spacing)
└── utils/        - ユーティリティ関数
```

## 📱 主要機能一覧

### 1. 散歩記録機能
- **日常散歩**: リアルタイムGPS記録、距離・時間表示、写真撮影/選択
- **おでかけ散歩**: 公式ルートに沿った記録、ピン投稿作成
- **統計表示**: 総距離71.9km、総散歩回数40回、総時間153分
- **散歩中バナー（Phase 4新機能）**: 進行中の散歩を画面下部に常時表示

### 2. ソーシャル機能
- **ピン投稿**: 位置情報 + 写真 + コメント
- **いいね & コメント**: 投稿への共感表現
- **フォロー/フォロワー**: ユーザー間の繋がり
- **通知**: いいね、コメント、フォロー、バッジ解除の通知

### 3. バッジシステム
- **自動解除**: 散歩記録完了時に条件チェック
- **バッジの種類**: 距離（5km, 10km, 50km等）、エリア探索、散歩回数、フォロワー数、ピン投稿数
- **特別バッジ**: Early Adopter（初期ユーザー）

### 4. 地図機能
- **HomeTab地図**: 最新ピン投稿の位置を表示
- **散歩記録画面**: 現在地マーカー、ルート描画、ズーム機能
- **MapTab**: 箱根エリア全体、ルート一覧、ピン投稿マーカー

### 5. UI/UX最適化（TestFlight準備）
- **Shimmerローディング**: 体感速度2倍向上
- **エラーハンドリング**: 再試行ボタン付き
- **画像最適化**: メモリ使用量20-30%削減
- **iOS権限設定**: カメラ、GPS、フォトライブラリ
